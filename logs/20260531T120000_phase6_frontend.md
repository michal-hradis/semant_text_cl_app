# Phase 6 — Frontend implementation

## What changed

### `frontend/src/types/api.ts`
- `TaskDefinition`: added sampling config fields (`calib_ratio_initial`, `calib_initial_count`,
  `calib_ratio_ongoing`, `repeat_probability`, `target_coverage`) and `current_multiplier`.
- New `UserReliabilityResponse` interface for IRR admin data.

### `frontend/src/services/api.ts`
- `submitAnnotations` now returns `{ points: Record<string, number> }`.
- New: `getGroundTruth(textId, taskIds)` — calibration feedback.
- New: `uploadGroundTruth(jsonl)`, `uploadLlmAnnotations(jsonl)` — admin uploads.
- New: `getAdminIrr()`, `recomputeIrr()` — reliability dashboard.

### `frontend/src/components/TaskSelectionDialog.vue`
- Task header now shows an orange **×N.N pts** multiplier badge when `current_multiplier > 1.0`.

### `frontend/src/pages/ClassificationPage.vue`
- After submitting annotations, if `calibration_task_ids` is non-empty, the page fetches
  ground-truth labels and shows a **CalibrationFeedback** panel (per-task submitted vs GT
  comparison with match/incorrect indicator). A "Continue" button advances to the next text.

### `frontend/src/pages/LeaderboardPage.vue`
- Added **Score** column to all leaderboard tables (sorted by score).
- Added **Reliability** column (shown as percentage).
- "My contributions" panel now shows score prominently alongside annotation count.
- Per-task rows are now sorted by score instead of count.
- Removed unused imports (`watch`, `TaskStats`).

### `frontend/src/pages/AdminPage.vue`
- New **Reliability** tab: shows `UserReliabilityResponse` table with pairwise agreement,
  Cohen's κ, Krippendorff's α; "Recompute" button triggers `POST /api/admin/irr/recompute`.
- **Import** tab expanded with GT upload (JSONL → `POST /api/admin/ground-truth`) and
  LLM upload (JSONL → `POST /api/admin/llm-annotations`) sections.
- Task editor dialog now includes **Sampling configuration** fields (all 5 config knobs).

### `backend/text_classifier/crud.py`
- `list_enabled_tasks` now computes and returns `current_multiplier` for each task
  (scarcity multiplier based on actual annotation coverage vs target).

### `backend/tests/test_text_classifier_crud.py`
- Added `test_list_enabled_tasks_includes_multiplier`: verifies multiplier is 2.0 when
  there are no user annotations (max scarcity).
- Added `test_gt_feedback_via_get_gt_for_text_tasks`: verifies GT map returned correctly.

### `frontend/openapi.json`
- Regenerated from live backend.

### `docs/api_reference.md`
- Added `MyStats` response shape, `LeaderboardEntry` schema, `UserReliabilityResponse`
  schema documentation.

## Decisions
- Calibration feedback is shown as a full-page overlay (not a modal dialog) to ensure
  users read it before proceeding.
- Multiplier badge is only shown when multiplier > 1.0 to avoid clutter on fully-covered tasks.
- GT and LLM upload sections are placed in the Import tab (alongside prompt import) since
  they are all data-import operations.
